# Todo-List 全栈应用 — 技术规格

## 1. Goal
构建一个功能完整的 Todo-List 网站，支持用户注册登录、任务 CRUD、分类标签、优先级、到期日、搜索过滤、拖拽排序，数据通过后端 API 持久化实现多设备同步。

## 2. Architecture
前后端分离：React + Vite 前端 SPA，Express + TypeScript 后端 REST API，Prisma ORM + SQLite 数据库。JWT 认证，bcrypt 密码哈希。前端通过 Vite proxy 代理 `/api` 到后端 `localhost:3001`。

## 4. API Contract

### Auth

#### POST /api/auth/register
```json
// Request
{ "email": "user@example.com", "password": "123456", "name": "Terry" }
// Response 201
{ "token": "jwt...", "user": { "id": 1, "email": "user@example.com", "name": "Terry" } }
// Error 409
{ "error": "Email already exists" }
```

#### POST /api/auth/login
```json
// Request
{ "email": "user@example.com", "password": "123456" }
// Response 200
{ "token": "jwt...", "user": { "id": 1, "email": "user@example.com", "name": "Terry" } }
// Error 401
{ "error": "Invalid credentials" }
```

#### GET /api/auth/me
```
Headers: Authorization: Bearer <token>
```
```json
// Response 200
{ "id": 1, "email": "user@example.com", "name": "Terry" }
```

## 5. Interface Signatures

### Prisma Schema (核心模型)

```prisma
enum Priority {
  LOW
  MEDIUM
  HIGH
}

model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  password  String
  name      String
  todos     Todo[]
  categories Category[]
  tags      Tag[]
  createdAt DateTime @default(now())
}

model Todo {
  id          Int       @id @default(autoincrement())
  title       String
  description String?
  completed   Boolean   @default(false)
  priority    Priority  @default(MEDIUM)
  dueDate     DateTime?
  position    Int       @default(0)
  userId      Int
  user        User      @relation(fields: [userId], references: [id])
  categoryId  Int?
  category    Category? @relation(fields: [categoryId], references: [id], onDelete: SetNull)
  tags        Tag[]
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}

model Category {
  id     Int    @id @default(autoincrement())
  name   String
  color  String @default("#6B7280")
  userId Int
  user   User   @relation(fields: [userId], references: [id])
  todos  Todo[]
}

model Tag {
  id     Int    @id @default(autoincrement())
  name   String
  userId Int
  user   User   @relation(fields: [userId], references: [id])
  todos  Todo[]
}
```

### Auth Middleware
```typescript
// server/src/middleware/auth.ts
export function authMiddleware(req: Request, res: Response, next: NextFunction): void
// 从 Authorization header 提取 JWT，验证后将 userId 挂到 req.userId
```

### Route Handlers (核心签名)
```typescript
// server/src/routes/auth.ts
POST /register (req: { email, password, name }) => { token, user }
POST /login    (req: { email, password }) => { token, user }
GET  /me       (req: authenticated) => user

// server/src/routes/todos.ts
GET    /        (req: authenticated, query filters) => { todos }
POST   /        (req: authenticated, body) => todo
PUT    /:id     (req: authenticated, body partial) => todo
DELETE /:id     (req: authenticated) => 204
PUT    /reorder (req: authenticated, { orderedIds }) => { message }

// server/src/routes/categories.ts
GET    / => { categories }
POST   / => category
PUT    /:id => category
DELETE /:id => 204

// server/src/routes/tags.ts
GET    / => { tags }
POST   / => tag
DELETE /:id => 204
```

## 6. Database Schema Changes
全新项目，无迁移。使用 `prisma migrate dev --name init` 初始化。

SQLite 文件路径: `server/prisma/dev.db`

索引:
- `Todo.userId` (自动，外键)
- `Todo.categoryId` (自动，外键)
- `User.email` (unique，自动)
- `Todo.position` (用于排序查询)

## 7. Negative Constraints

❌ 禁止在前端硬编码后端 URL，必须通过 Vite proxy `/api`
❌ 禁止明文存储密码，必须用 bcrypt hash
❌ 禁止在 JWT payload 中放密码
❌ 禁止不验证 JWT 就访问受保护路由
❌ 禁止跨用户访问数据（所有查询必须带 userId 过滤）
❌ 禁止在 Todo 的 DELETE/PUT 中不校验 ownership
❌ 禁止使用 `any` 类型（TypeScript strict mode）
❌ 禁止不处理 Prisma 错误（如 unique constraint violation）

## 8. Task Decomposition

### 后端任务 (Codex)

1. **初始化项目**: `server/package.json` + `server/tsconfig.json` + 依赖安装
2. **Prisma Schema**: `server/prisma/schema.prisma` + `prisma migrate dev`
3. **Express 入口**: `server/src/index.ts` — cors, json parser, 路由挂载, 错误处理
4. **Auth 中间件**: `server/src/middleware/auth.ts` — JWT 验证
5. **Auth 路由**: `server/src/routes/auth.ts` — register, login, me
6. **Todo 路由**: `server/src/routes/todos.ts` — CRUD + reorder + 搜索过滤
7. **Category 路由**: `server/src/routes/categories.ts` — CRUD
8. **Tag 路由**: `server/src/routes/tags.ts` — CRUD

### 前端任务 (Claude)

1. **初始化项目**: Vite + React + TS + Tailwind 脚手架
2. **API 层**: axios client + auth/todos/categories/tags API
3. **Auth 上下文**: AuthContext + useAuth hook
4. **认证页面**: LoginPage + RegisterPage
5. **Layout**: 全局布局 + 导航 + 侧边栏
6. **Todo 核心**: TodoList + TodoItem + TodoForm + useTodos hook
7. **分类/标签**: CategorySidebar + 标签管理
8. **搜索过滤**: FilterBar
9. **拖拽排序**: @dnd-kit 集成到 TodoList
10. **根目录**: workspace package.json + 并发启动脚本

## 9. Complexity Verdict

**Medium** — 前后端分离全栈应用，功能模块多但每个模块逻辑清晰。Codex 后端 + Claude 前端并行。
